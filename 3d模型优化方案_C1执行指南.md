# 3D 模型优化方案（可适配当前项目）
## 计划 C1 执行指南

## 1. 背景与目标

当前场景中的 GLB 模型总大小约 **785 MB**，可能导致：

- 首屏加载慢
- 运行帧率低
- 浏览器内存压力大，甚至崩溃

项目前端已接入 **DRACOLoader**，因此可直接使用 Draco 压缩后的模型文件，**无需额外改动解码逻辑**。

---

## 2. 执行顺序（C1）

### 第一步：安装 `gltf-transform`（一次性）

可在项目根目录或 `frontend` 目录执行：

```bash
npm install -g @gltf-transform/cli
```

或使用 `npx`（不全局安装）：

```bash
npx @gltf-transform/cli --help
```

> 若使用 `npx`，下文中所有 `gltf-transform` 命令都写为 `npx @gltf-transform/cli`。

---

### 第二步：Draco 压缩（优先执行）

#### 2.1 单文件示例

```bash
cd frontend
npx @gltf-transform/cli draco assets/models/2030_projector_02.glb assets/models/2030_projector_02_draco.glb --method edgebreaker
```

#### 2.2 批量处理（PowerShell）

```powershell
cd frontend
Get-ChildItem -Path "assets/models" -Filter "*.glb" | ForEach-Object {
  $out = $_.FullName -replace '\.glb$','_draco.glb'
  npx @gltf-transform/cli draco $_.FullName $out --method edgebreaker
}
```

会生成 `*_draco.glb` 文件。

#### 2.3 接入方式（两选一）

- **方式 A：替换原文件**
  - 将 `xxx_draco.glb` 改名回原文件名，覆盖原始模型。
- **方式 B：修改路径配置**
  - 在 `ThreeScene.tsx` 中将 `ITEM_CONFIGS` / `HISTORIC_MODEL_BY_SLOT` 的路径改为指向 `*_draco.glb`。

#### 2.4 预期收益

典型情况下，体积可下降 **60%–80%**（例如 `80MB → 约 16–25MB`）。

---

### 第三步：贴图缩小（可选，进一步减重）

如果模型内贴图过大，可先限制分辨率，再进行 Draco 压缩。

#### 3.1 单文件流程

```bash
npx @gltf-transform/cli resize input.glb resized.glb --width 1024 --height 1024
npx @gltf-transform/cli draco resized.glb output.glb --method edgebreaker
```

#### 3.2 批量流程（PowerShell）

```powershell
cd frontend
Get-ChildItem -Path "assets/models" -Filter "*.glb" | ForEach-Object {
  $resized = $_.FullName -replace '\.glb$','_resized.glb'
  $out = $_.FullName -replace '\.glb$','_opt.glb'
  npx @gltf-transform/cli resize $_.FullName $resized --width 1024 --height 1024
  npx @gltf-transform/cli draco $resized $out --method edgebreaker
  Remove-Item $resized -ErrorAction SilentlyContinue
}
```

输出文件为 `*_opt.glb`，可按需替换原文件或修改配置路径。

---

### 第四步：确认前端 Draco 支持（本项目已完成补齐）

本次实操中发现：若未正确接入 `DRACOLoader`，压缩后模型会加载失败并出现“方块占位”。

本项目现已在 `ThreeScene.tsx` 完成以下配置：

- 引入并初始化 `DRACOLoader`
- `dracoLoader.setDecoderPath('https://www.gstatic.com/draco/versioned/decoders/1.5.7/')`
- `gltfLoader.setDRACOLoader(dracoLoader)`
- 组件销毁时执行 `dracoLoader.dispose()`

因此，当前代码已可稳定加载 Draco 压缩后的 `.glb` 文件。

---

## 3. 建议落地策略（适配当前项目）

1. 先对 `frontend/assets/models` 全量跑一轮 Draco，产出 `*_draco.glb`。
2. 优先替换体积最大的模型，验证页面加载与交互稳定性。
3. 若仍偏大，再对问题模型执行 `resize(1024) + draco`，产出 `*_opt.glb`。
4. 最后统一命名与路径策略（覆盖原文件或集中改配置），避免混用。
